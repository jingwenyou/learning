一、核心方法论
================================================================================

1.1 性能分析三大步骤
--------------------
   1. 转向(Advection): 从上到下分析，从用户到系统底层
   2. 消除(Removal): 逐步排除非瓶颈因素
   3. 热点定位(Hot Spot): 找到资源瓶颈所在

1.2 性能指标(USE方法)
--------------------
   - Utilization (利用率): 资源忙碌时间百分比
   - Saturation (饱和度): 资源排队/等待工作量
   - Errors (错误率): 错误发生频率

1.3 延迟分析
------------
   - 定义延迟类型: 初始化延迟、服务延迟、传输延迟
   - 使用直方图和百分位数(50th, 90th, 99th, 99.9th)

1.4 静态性能调优
----------------
   - 检查系统架构设计
   - 检查系统默认配置
   - 消除不必要的资源使用

二、CPU性能工具
================================================================================

2.1 观测工具
------------
   * sar (System Activity Reporter)
     - sar -u: CPU利用率
     - sar -q: 运行队列长度
     - sar -P ALL: 每个CPU核心详情
     示例: sar -u 1 5 (每秒采样，共5次)

   * mpstat: 多处理器统计
     - mpstat -P ALL: 所有CPU核心统计
     - %idle, %iowait, %sys等指标

   * top/htop: 实时进程监控
     - top: 按CPU排序查看高消耗进程
     - htop: 更友好的交互界面

   * pidstat: 进程级CPU统计
     - pidstat -p <pid> 1: 单个进程CPU使用

   * perf: Linux性能分析工具
     - perf top: 热点函数分析
     - perf record -g: 采样记录(用于火焰图)
     - perf sched: 调度器分析

   * turbostat: Intel Turbo Boost监控
     - 查看CPU频率和功耗

2.2 调优工具
------------
   * cpufreq-info/cpupower: CPU频率控制
   * taskset: 绑核处理
   * numactl: NUMA亲和性设置

2.3 热点分析
------------
   * 火焰图(Flame Graph)
     - 用 perf/DTrace 采样生成
     - 类型: CPU火焰图、内存火焰图、off-CPU火焰图
     - 工具: Brendan Gregg的 flamegraph.pl

   * 热力图(Heat Maps)
     - CPU使用率分布
     - 延迟分布

三、内存性能工具
================================================================================

3.1 观测工具
------------
   * free: 内存使用概况
     - free -m: MB为单位显示

   * vmstat: 虚拟内存统计
     - vmstat 1: 每秒刷新
     - si/so: 交换入/交换出

   * sar -r: 内存利用率
   * slabtop: 内核Slab缓存分析

   * pmap: 进程内存映射
     - pmap -x <pid>: 详细内存地址

   * smem: 物理内存使用(可按比例显示)

   * numastat: NUMA节点内存统计

3.2 调优参数
------------
   - /proc/sys/vm/swappiness: 交换倾向(0-100)
   - /proc/sys/vm/dirty_ratio: 脏页比例
   - /proc/sys/vm/vfs_cache_pressure: 文件缓存回收压力

3.3 内存分析
------------
   * memleak: 内存泄漏检测( BCC工具集)
   * memcg: cgroup内存统计

四、磁盘/存储性能工具
================================================================================

4.1 观测工具
------------
   * iostat: I/O统计(最常用)
     - iostat -xz 1: 设备级I/O详情
     - r/s, w/s: 读写次数每秒
     - rkB/s, wkB/s: 读写速度
     - await: 平均I/O响应时间
     - %util: 设备利用率(接近100% = 饱和)

   * sar -d: 磁盘设备活动
   * iotop: 进程级I/O监控
   * ioping: I/O延迟测试
   * blktrace: 块设备I/O追踪

   * df/du: 文件系统使用量
     - df -h: 人性化显示
     - du -sh *: 目录大小

4.2 调优工具
------------
   - hdparm: SATA磁盘参数调整
   - sdparm: SCSI/SAS磁盘参数
   - blockdev: 块设备参数

4.3 文件系统
------------
   * ext4:
     - tune2fs: 调整参数
     - dumpe2fs: 查看超级块信息

   * XFS:
     - xfs_info: 文件系统信息
     - xfs_growfs: 扩展文件系统

   * btrfs:
     - btrfs filesystem: 管理命令

4.4 性能分析
------------
   * fio: 磁盘I/O基准测试
     - fio --name=randread --rw=randread --bs=4k --numjobs=4
   * bonnie++: 文件系统基准测试
   * iozone: 多种I/O模式测试

五、网络性能工具
================================================================================

5.1 观测工具
------------
   * netstat/ss: 套接字统计
     - ss -s: 汇总统计
     - ss -tanp: 所有TCP连接
     - ss -u -anp: UDP统计

   * sar -n: 网络统计
     - sar -n DEV 1: 设备统计
     - sar -n TCP,ETCP 1: TCP统计

   * nicstat: 网卡流量统计
   * iftop: 实时流量监控
   * nethogs: 按进程流量

   * ip neigh: ARP表
   * ip route: 路由表
   * ip -s link: 链路层统计

5.2 网络延迟测试
----------------
   * ping: 基础延迟测试
   * traceroute/mtr: 路由跳数分析
   * tcping: TCP延迟测试
   * iperf3: 带宽基准测试

5.3 调优工具
------------
   * sysctl: 内核网络参数
     - net.core.rmem_max: 接收缓冲区
     - net.core.wmem_max: 发送缓冲区
     - net.ipv4.tcp_tw_reuse: TCP重用

   * tc: 流量控制(QoS)
   * ip: 网络配置

5.4 网络抓包分析
----------------
   * tcpdump: 原始数据包捕获
   * wireshark/tshark: 协议分析
   * ngrep: 网络层grep
   * zeek: 网络安全监控

六、系统级综合工具
================================================================================

6.1 观测工具
------------
   * dstat: 综合系统统计(可替代vmstat/iostat/sar)
     - dstat -cdny 1

   * glances: 全方位监控
   * htop/top: 实时进程监控
   * atop: 高级系统/进程监控

   * /proc系统:
     - /proc/cpuinfo: CPU信息
     - /proc/meminfo: 内存信息
     - /proc/diskstats: 磁盘统计
     - /proc/net/dev: 网络设备统计

6.2 追踪工具(Tracing)
---------------------
   * strace: 系统调用追踪
     - strace -p <pid>: 追踪进程
     - strace -c: 统计系统调用

   * ltrace: 库函数调用追踪
   * gdb: 调试器

   * ftrace: Linux内核追踪
     - /sys/kernel/debug/tracing

   * perf: Linux性能采样器

   * BCC工具集(eBPF):
     - opensnoop: 文件打开追踪
     - execsnoop: 进程执行追踪
     - biosnoop: 磁盘I/O追踪
     - tcplife: TCP连接生命周期
     - runqlat: 运行队列延迟

   * bpftrace: eBPF高级追踪语言

6.3 DTrace (动态追踪)
---------------------
   * Solaris/Linux性能追踪标配
   * Brendan Gregg开发的很多工具基于此

十二、实战分析流程
================================================================================

12.1 性能问题排查步骤
---------------------
   Step 1: 确定问题现象
   - 用户投诉: 慢、卡、无响应

   Step 2: 检查系统概况
   - dstat -cdny 1 (CPU/磁盘/网络/内存综合)

   Step 3: 定位资源瓶颈
   - CPU: sar -u, top
   - 内存: free -m, vmstat 1
   - 磁盘: iostat -xz 1
   - 网络: sar -n DEV 1

   Step 4: 追踪热点
   - perf top (CPU热点)
   - strace -c (系统调用)
   - 火焰图分析

   Step 5: 验证修复
   - 重复Step 2-4观察变化

12.2 常见瓶颈及解决
------------------
   CPU高:
   - 优化算法/代码
   - 多线程/多进程
   - CPU绑核

   内存高:
   - 释放缓存: sync; echo 3 > /proc/sys/vm/drop_caches
   - OOM-killer: 检查dmesg
   - 内存泄漏: memleak/bpftrace

   磁盘I/O高:
   - SSD升级
   - RAID优化
   - 文件系统选择(ext4/xfs)
   - 应用程序I/O模式优化

   网络延迟:
   - TCP参数调优
   - 内核网络栈优化
   - 拥塞控制算法调整

12.3 自动化监控方案
-------------------
   推荐组合:
   - 指标采集: Prometheus + node_exporter
   - 日志收集: fluent-bit + ElasticSearch
   - 可视化: Grafana
   - 告警: AlertManager
   - 追踪: Jaeger/Zipkin

十三、关键参考指标阈值
================================================================================

   CPU:
   - %user > 80%: 可能需要更多CPU资源
   - %sys > 30%: 内核开销过高
   - runq-sz > 10: 运行队列过长

   内存:
   - %memused > 90%: 内存压力
   - %swapped > 0%: 正在使用交换空间
   - vmstat的si/so持续非零: 内存不足

   磁盘:
   - %util > 80%: I/O饱和
   - await > 10ms: 响应时间过长
   - avgqu-sz > 4: 队列过长

   网络:
   - %ifutil > 80%: 带宽饱和
   - 重传率 > 1%: 网络问题
   - 连接数过多: 可能是攻击或泄露
