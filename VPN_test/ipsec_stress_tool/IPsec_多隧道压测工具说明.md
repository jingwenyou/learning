# IPsec 多隧道并发压测工具

## 概述

本工具用于测试 IPsec 多隧道场景下的并发收发包能力，支持从多个源IP同时向目标IP发送数据包，并统计延迟、丢包率和吞吐量。

## 原理说明

### 架构

```
┌─────────────────────────┐                    ┌─────────────────────────┐
│   192.168.110.240       │                    │   192.168.110.199       │
│  ┌───────────────────┐  │                    │  ┌───────────────────┐  │
│  │ ipsec_multi_      │  │                    │  │ ipsec_echo_       │  │
│  │ tunnel_test.py    │  │ ════════════════>  │  │ server.py         │  │
│  │                   │  │   IPsec 隧道       │  │                   │  │
│  │ [Sender线程 x 64] │  │ <════════════════  │  │ [回包服务]        │  │
│  │ [Receiver线程 x 64]│ │   UDP Echo         │  │ 监听 7000-7063    │  │
│  └───────────────────┘  │                    │  └───────────────────┘  │
└─────────────────────────┘                    └─────────────────────────┘
```

### 测试流程

1. **发包阶段**: 每个源IP启动一个 Sender 线程，按指定速率发送 UDP 数据包
   - 源IP: 从 `enp1s0f0` 网卡获取的所有IP（共64个）
   - 目的IP: 10.3.3.199
   - 目的端口: 7000+i (每个源IP对应一个端口)
   - 数据包包含: 时间戳(seq) + 序列号

2. **回包阶段**: 对端 echo_server 收到包后立即原路返回

3. **收包阶段**: 每个源IP启动一个 Receiver 线程，接收回包并计算延迟
   - 延迟 = 收到回包时间 - 发送时间
   - 丢失 = 发送数 - 接收数

### 延迟异常说明

延迟异常高（500+秒）的原因分析：
- 回包在 IPsec 隧道或网络设备中排队积压
- 设备处理能力不足导致 buffer 堆积
- 往返路径不一致造成乱序

这不是工具问题，建议：
1. 在 10.3.3.199 上用 `tcpdump` 抓包观察
2. 降低发包速率 `-r` 参数
3. 检查隧道带宽和设备负载

## 快速开始

### 1. 启动回包服务（仅需一次）

```bash
# 在 192.168.110.199 上启动
ssh haitiai@192.168.110.199
python3 /tmp/ipsec_echo_server.py
```

服务启动后自动监听 7000-7063 端口，收到数据包后自动回传。

### 2. 运行压测

```bash
# 在 192.168.110.240 上运行
python3 /tmp/ipsec_multi_tunnel_test.py -d 10.3.3.199 -i enp1s0f0 -r 100
```

按 `Ctrl+C` 停止测试。

## 发包工具参数说明

```bash
python3 /tmp/ipsec_multi_tunnel_test.py [参数]
```

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `-d DST` | 目标IP | 10.3.3.199 |
| `-i IFACE` | 源网卡，从该网卡获取所有IP | enp1s0f0 |
| `-t TIME` | 持续时间（秒），0=无限 | 0 |
| `-r RATE` | 每IP每秒发包数 | 100 |
| `-s SIZE` | 数据包大小（字节） | 1024 |
| `-p PROTO` | 协议类型 udp/tcp | udp |

### 使用示例

```bash
# 基础测试：每IP 100pps，持续运行
python3 /tmp/ipsec_multi_tunnel_test.py -d 10.3.3.199 -i enp1s0f0 -r 100

# 限时测试：运行1小时
python3 /tmp/ipsec_multi_tunnel_test.py -d 10.3.3.199 -i enp1s0f0 -t 3600 -r 200

# 大包测试：512字节
python3 /tmp/ipsec_multi_tunnel_test.py -d 10.3.3.199 -i enp1s0f0 -r 100 -s 512

# TCP协议
python3 /tmp/ipsec_multi_tunnel_test.py -d 10.3.3.199 -i enp1s0f0 -r 100 -p tcp
```

## 故障排查

### 问题：收包为0

1. 确认回包服务正在运行：
   ```bash
   ssh haitiai@192.168.110.199 "ps aux | grep ipsec_echo"
   ```

2. 确认回包服务端口绑定正常：
   ```bash
   ssh haitiai@192.168.110.199 "netstat -ulnp | grep python"
   ```

3. 在服务端抓包确认收到数据：
   ```bash
   ssh haitiai@192.168.110.199 "tcpdump -i any -n udp port 7000-7063"
   ```

4. 检查IPsec隧道状态和路由

### 问题：部分IP收包为0

这通常是IPsec隧道配置问题，不是工具问题。检查：
- 该IP是否已配置IPsec隧道
- 路由是否正确
- 隧道是否激活

### 问题：丢包率过高

1. 检查网络负载
2. 降低发包速率 `-r` 参数
3. 检查对端处理能力
