## 三、残留BUG (BUG-2/4/6/7)

### BUG-7 [严重度: 高] — ListenDrops 触发错误诊断路径
- **位置**: `health.py` L508-511
- **问题**: `listen_drops > 0` 时 `issues.append('高错误率')`，导致建议"检查网线和物理连接"
- **根因**: `DIAGNOSTIC_ADVICE['network']['Listen队列溢出']` 键已存在但未被调用
- **修复**: 改为 `issues.append('Listen队列溢出')`，5行代码即可

### BUG-6 [严重度: 中] — 网卡带宽 fallback 固化为 100Mbps
- **位置**: `system.py` L736-738
- **问题**: ethtool 失败(容器/VM常见)时默认 100Mbps，实际通常 1Gbps/10Gbps，误差10-100倍
- **修复**: 优先读 `/sys/class/net/<iface>/speed`（无需特权，比 ethtool 可靠）

### BUG-2 [严重度: 中] — context_switches/interrupts 为累计绝对值
- **位置**: `system.py` L174-178
- **问题**: 长期运行系统的 ctxt 达数十亿，直接展示无参考意义
- **修复**: 新增 `get_scheduler_rate(interval=1)` 双采样，输出 ctx_switches/sec

### BUG-4 [严重度: 低] — 低IOPS时 await 数量级误差
- **位置**: `system.py` L577-580
- **问题**: `interval=0.5` 内 I/O 次数 < 2 时 await 误差可达数量级
- **修复**: 添加最小样本保护 `if total_ios < 5: skip await reporting`

## 五、《性能之巅》第1版 vs 第2版覆盖

### 第1版: 56.7% (17/30 主题)

**未覆盖 (高价值)**:
- CPU 调度延迟分布 (`/proc/schedstat` 或 BPF runqlat)
- CPU 中断按类型分解 (`/proc/interrupts`)
- 内存 NUMA 节点分布 + 跨节点访问延迟
- 磁盘 SMART 健康预警 (`smartctl`)
- 网络 UDP 错误 (InErrors/RcvbufErrors)

### 第2版 BPF/eBPF: 33.3% (4/12 主题)

**已覆盖的第2版内容**: PSI、页面扫描早期告警、TCP高级指标、连接状态精确解析

**天然受限**: BPF 工具 (runqlat/biolatency/offcputime/profile) 需要 CAP_BPF 或 root 权限
以及 BCC/bpftrace 依赖，CLI 工具架构下无法直接集成；这是设计边界，非缺陷。
