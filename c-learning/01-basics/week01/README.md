## 二、C 程序的生命周期

你写的 `.c` 文件到能运行，经历四步：

```
源码 (.c)
  │
  ▼  预处理（展开 #include、#define）
  │
  ▼  编译（C → 汇编）
  │
  ▼  汇编（汇编 → 机器码 .o）
  │
  ▼  链接（.o + 库 → 可执行文件）
  │
  ▼
可执行文件
```

平时用 `gcc hello.c -o hello` 一步完成全部四步。

**对比 Python：** Python 是解释执行，不需要这个过程。C 编译后直接是机器码，速度快，但必须先编译。

## 三、基本数据类型

### 整数类型

```c
char    c = 'A';    // 1 字节，-128 ~ 127（有符号）或 0 ~ 255（无符号）
int     i = 42;     // 4 字节，-2147483648 ~ 2147483647
long    l = 123L;   // 8 字节（64位Linux）
```

**记住：** `sizeof(int)` 不是固定的，依赖平台。用 `%zu` 打印 sizeof 的结果。

### 浮点类型

```c
float   f = 3.14f;   // 4 字节，精度约 6-7 位有效数字
double  d = 3.14;    // 8 字节，精度约 15-16 位有效数字
```

**对比 Python：** Python 的 `float` 对应 C 的 `double`，Python 没有单精度浮点。

### printf 格式符速查

| 类型 | 格式符 |
|------|--------|
| int | `%d` |
| unsigned int | `%u` |
| long | `%ld` |
| float | `%f` |
| double | `%f` 或 `%.15f` |
| char | `%c`（字符）`%d`（ASCII值）|
| 十六进制 | `%x` 或 `%X` |
| sizeof 结果 | `%zu` |

## 四、两个重要陷阱

### 陷阱 1：有符号整数溢出

```c
int x = 2147483647;  // INT_MAX
x = x + 1;          // 未定义行为！编译器警告，结果变成 -2147483648
```

**为什么：** int 用二进制补码存储，最高位是符号位，溢出时符号位翻转。

```
 0111...1111  (INT_MAX = 2147483647)
+          1
-----------
 1000...0000  (变成了 -2147483648)
```

### 陷阱 2：无符号回绕（合法行为）

```c
unsigned int u = 0;
u = u - 1;   // 合法！结果是 4294967295 (UINT_MAX)
```

**为什么：** 无符号整数的算术是模 2^32 运算，0 - 1 回绕到最大值。

## 五、编译参数说明

```bash
gcc -Wall -Wextra -std=c11 -g -o 输出文件 源文件.c
```

| 参数 | 含义 |
|------|------|
| `-Wall` | 开启常见警告 |
| `-Wextra` | 开启额外警告 |
| `-std=c11` | 使用 C11 标准 |
| `-g` | 包含调试信息（gdb 需要） |
| `-o hello` | 指定输出文件名 |

**原则：** 始终用 `-Wall -Wextra`，把所有警告当错误处理。
