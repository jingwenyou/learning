# 02 安全测试用例 —— 可复用的武器库

> 把安全检查变成**标准化的测试用例**，这样每次测试都不会遗漏。
> 以下是按功能分类的安全测试用例模板，直接拿来用。

---

## 一、通用安全测试用例模板

### 认证模块

```
TC-AUTH-001: 暴力破解防护
  前置条件：已知一个有效用户名
  步骤：连续20次输入错误密码
  预期：出现验证码/账户锁定/限频（429状态码）
  级别：高

TC-AUTH-002: 用户名枚举
  步骤：分别用存在和不存在的用户名登录
  预期：错误提示完全相同（"用户名或密码错误"）
  关注：响应内容、长度、时间、状态码 全部应一致
  级别：中

TC-AUTH-003: 弱密码
  步骤：注册/修改密码为 "123456"、"password"
  预期：被拒绝
  级别：中

TC-AUTH-004: Session固定
  步骤：记录登录前后的Session ID
  预期：登录后Session ID重新生成
  级别：高

TC-AUTH-005: 登出有效性
  步骤：登出 → 用旧Session ID/Token发请求
  预期：请求被拒绝（401）
  级别：高

TC-AUTH-006: Cookie安全属性
  步骤：登录 → 检查Set-Cookie响应头
  预期：HttpOnly ✓, Secure ✓, SameSite=Lax或Strict ✓
  级别：中

TC-AUTH-007: 密码传输安全
  步骤：抓包观察登录请求
  预期：使用HTTPS，密码不在URL中
  级别：高
```

### 授权模块

```
TC-AUTHZ-001: 水平越权（查看）
  前置条件：两个普通用户账号A和B
  步骤：用A的Token请求B的资源（修改URL中的ID）
  预期：返回403或404
  级别：高

TC-AUTHZ-002: 水平越权（修改）
  步骤：用A的Token修改B的数据（PUT/PATCH请求，改目标ID）
  预期：返回403
  级别：严重

TC-AUTHZ-003: 水平越权（删除）
  步骤：用A的Token删除B的数据
  预期：返回403
  级别：严重

TC-AUTHZ-004: 垂直越权
  步骤：用普通用户Token访问管理接口（/admin/*, /api/admin/*）
  预期：返回403
  级别：严重

TC-AUTHZ-005: 未认证访问
  步骤：不带Token/Cookie直接访问需要登录的接口
  预期：返回401
  级别：高
```

### 输入验证模块

```
TC-INPUT-001: SQL注入
  步骤：在参数中输入 ' " OR 1=1 -- ; AND SLEEP(3)
  预期：不报SQL错误，不返回额外数据，无延时
  级别：严重

TC-INPUT-002: XSS
  步骤：在输入中提交 <script>alert(1)</script>
  步骤：查看输出页面源码
  预期：特殊字符被HTML编码（< 变成 &lt;）
  级别：高

TC-INPUT-003: 命令注入
  步骤：在接受系统操作的参数中输入 ; whoami | id
  预期：不执行系统命令
  级别：严重

TC-INPUT-004: 路径遍历
  步骤：在文件相关参数中输入 ../../etc/passwd
  预期：不返回系统文件内容
  级别：高
```

### CSRF模块

```
TC-CSRF-001: CSRF Token存在性
  步骤：查看敏感操作的请求中是否有Token参数
  预期：存在csrf_token或类似参数
  级别：中

TC-CSRF-002: CSRF Token有效性
  步骤：删除/篡改Token后提交请求
  预期：请求被拒绝
  级别：高
```

### 文件上传模块

```
TC-UPLOAD-001: 危险文件类型
  步骤：上传 .php/.jsp/.asp 文件
  预期：被拒绝
  级别：严重

TC-UPLOAD-002: Content-Type绕过
  步骤：上传PHP文件但Content-Type设为image/jpeg
  预期：被拒绝
  级别：高

TC-UPLOAD-003: 文件大小限制
  步骤：上传超大文件（100MB+）
  预期：被拒绝，不影响服务可用性
  级别：中

TC-UPLOAD-004: 文件名安全
  步骤：文件名中包含 ../、特殊字符
  预期：文件名被净化，不产生路径遍历
  级别：高
```

### 信息泄露模块

```
TC-INFO-001: 错误页面
  步骤：触发404/500错误
  预期：不暴露技术栈、路径、SQL语句、堆栈信息
  级别：中

TC-INFO-002: 安全响应头
  步骤：检查HTTP响应头
  预期：存在 X-Frame-Options, CSP, X-Content-Type-Options, HSTS
  级别：中

TC-INFO-003: 敏感路径
  步骤：访问 /.git/config, /.env, /robots.txt, /swagger-ui.html
  预期：敏感路径不可访问或不泄露信息
  级别：高
```

## 二、使用建议

```
[1] 每个项目挑选适用的用例
  不是所有用例都适用于所有项目
  根据项目功能挑选相关用例

[2] 新功能重点测
  已有功能做回归
  新开发的功能做全面安全测试

[3] 发现新漏洞模式就补充用例
  这个库应该持续增长
  每次发现一个新的漏洞类型，就加一条用例

[4] 和功能测试用例一起管理
  在同一个测试管理系统中
  安全用例也是测试用例的一部分
```
