# C 语言学习计划

> 背景：测试工程师，非计算机专业，有 Python 基础，日常测试网络/安全/密码设备
> 目标：通过 C 深入理解计算机底层，提升职业深度
> 总周期：约 7 个月（每天 1-2 小时）

---

## 核心原则

1. **指针没搞懂，不往后走**。宁可多花两周，也不带着糊涂往前赶
2. **每写一个函数，同时写测试**。你是测试工程师，用测试驱动学习
3. **卡住超过 3 天，退回去用更简单的例子重新理解**，不要硬啃
4. **理论穿插在实践中**，不集中啃理论书

## 第一阶段：C 语言基础（第 1-8 周）

### 学习资源
- 主线书籍：《C Programming: A Modern Approach》（K.N. King）
- 视频配合：CS50x 前半部分（哈佛公开课，免费）
- 指针可视化工具：https://pythontutor.com/c.html （每次写指针代码都先在这里跑一遍）

### 周计划

| 周 | 主题 | 练习目标 | 验证标准 |
|----|------|---------|---------|
| 1 | 环境搭建 + 第一个程序 | 安装 gcc/gdb/valgrind/make；写 hello world；试 `int`/`char`/`float` | 能独立用 `gcc -Wall -Wextra` 编译运行，理解编译报错 |
| 2 | 控制流 + 函数 | if/for/while 练习；写 3-5 个小函数 | 能拆分多函数，理解声明与定义 |
| 3 | 数组 + 预处理器 | 冒泡排序、二分查找；`#define`/`#include` 守卫/条件编译 | 理解头文件保护机制，能写简单宏 |
| 4 | 指针（一） | 指针与地址、指针与数组；**在纸上画内存格子图** | 能画出内存图解释 `int *p = &a`，在 pythontutor 上验证 |
| 5 | 指针（二） | 指针运算、指针与函数参数、指针与数组关系 | 能用指针交换两个变量，理解值传递 vs 地址传递 |
| 6 | 字符串 + enum/typedef | 手写 strlen/strcpy/strcmp；用 `enum` 定义状态 | 理解字符串是 `char[]` + `\0`；能用枚举表达有限状态 |
| 7 | 结构体 + const/static | 结构体定义、嵌套、指针；`const` 正确性；`static` 作用域控制 | 理解 `const char *` vs `char * const`；知道 `static` 函数的意义 |
| 8 | 动态内存 | malloc/free 练习，链表（增删查） | valgrind 检查无内存泄漏 |

### 工程素养同步培养
- 第 1 周起：所有代码用 `gcc -Wall -Wextra` 编译，消除所有警告
- 第 2 周起：**每写一个函数，在 main 里写 assert 测试它**
- 第 3 周起：学写 Makefile（从最简单的开始）
- 第 5 周起：用 `gdb` 单步调试，观察变量和内存

### 自测宏（从第 2 周起贯穿使用）
```c
#include <assert.h>
#include <stdio.h>

#define TEST(name) printf("  TEST %s ... ", #name);
#define PASS()     printf("PASS\n");

// 用法示例
TEST(my_strlen_basic)
assert(my_strlen("hello") == 5);
assert(my_strlen("") == 0);
PASS()
```

## 第二阶段：实战项目 + 穿插理论（第 9-16 周）

> 从这个阶段开始，理论不再单独学，而是**遇到什么学什么**，穿插在项目中。

### 项目一：命令行工具集（第 9-10 周）
**做什么：**
- 写一个命令行选项解析器（处理 `-v`/`--help`/`-o filename` 等）
- 用它做一个简单的文件处理工具（统计行数/字数，类似 `wc`）

**学到什么：** 参数处理、错误路径、多文件组织（`.h` + `.c` 分离）
**测试方法：** 为每个函数写断言；测边界情况（空文件、超长行、无参数）
**验证：** 与系统 `wc` 对比输出

### 项目二：简易密码工具（第 11-13 周）⭐ 与工作关联
**做什么：**
- Caesar 加密/解密
- XOR 加密/解密
- 简单的 hex dump 工具（类似 xxd）

**学到什么：** 位运算、字节操作、文件 I/O
**测试方法：** 加密再解密，断言与原文一致；fuzzing 随机输入不崩溃
**验证：** 处理二进制文件不出错

**穿插理论 —— CSAPP 第 2 章（信息的表示）：**
- 写密码工具时你会遇到：整数溢出、有符号/无符号转换、字节序
- 边写边读第 2 章对应内容，立刻能理解为什么

### 项目三：TCP echo server（第 14-16 周）⭐ 与工作关联
**做什么：**
- 用 socket API 写一个 TCP 回显服务器
- 配套写一个客户端测试工具
- 支持多个客户端连接（用 `fork` 或 `select`）

**学到什么：** socket 编程、网络字节序、阻塞 I/O、进程基础
**测试方法：** 客户端发什么，服务端原样返回；测并发连接；测异常断开
**验证：** 用你的 Python 脚本作为客户端交叉验证

**穿插理论 —— CSAPP 第 8 章（进程）：**
- 写 `fork` 处理并发时，读第 8 章理解进程是什么
- 用 `strace` 跟踪你的 server 的系统调用

### 工程素养同步培养
- 头文件与源文件分离（`.h` + `.c`）
- 每个项目写简单的 README
- 用 `git` 管理代码，每个功能一次提交
- 学习 `strace` 跟踪系统调用

## 第三阶段：计算机系统（第 17-22 周）

### 学习资源
- 主线书籍：《深入理解计算机系统》（CSAPP）
- 第 2、8 章已在第二阶段穿插学过，这里学剩余重点章节

### 章节顺序（按优先级）

| 顺序 | 章节 | 核心收获 | 与你工作的关联 |
|------|------|---------|--------------|
| 1 | 第 3 章：机器级表示 | C 代码如何变成汇编/机器指令 | 做 Bomb Lab 的前置知识；理解安全漏洞原理 |
| 2 | 第 7 章：链接 | 编译→汇编→链接→可执行文件全过程 | 理解库的加载、符号解析 |
| 3 | 第 11 章：网络编程 | 系统级网络 API 全貌 | ⭐ 直接关联网络设备测试 |
| 4 | 第 9 章：虚拟内存 | 地址空间隔离、页表、内存保护 | ⭐ 理解安全设备的隔离机制 |
| 5 | 第 6 章：存储器层次 | 缓存原理 | 选读，优先级最低 |

### 配套实验

| 实验 | 优先级 | 收获 |
|------|--------|------|
| Bomb Lab | **必做** | 用 gdb 逆向分析汇编 —— 对安全测试直接有用 |
| Proxy Lab | **必做** | 网络编程综合练习 ⭐ 比 Malloc Lab 更贴合你的方向 |
| Data Lab | 推荐 | 理解位运算和数据表示 |
| Malloc Lab | 选做 | 深入理解内存管理 |

### 工程素养同步培养
- 学习 `objdump` 查看编译产物
- 理解 ELF 文件格式基础
- 用 `gdb` 阅读汇编级代码

## 第四阶段：综合项目 + 进阶（第 23-30 周）

### 综合项目：简易抓包与对比工具 ⭐⭐ 与工作高度关联
**做什么：** 极简版 tcpdump + 包对比功能
1. 用 raw socket 捕获网络包
2. 解析以太网帧头、IP 头、TCP/UDP 头
3. 按条件过滤（端口、协议）
4. 格式化输出
5. **对比两个抓包结果是否一致**（贴合你的测试场景）

**阶段拆分：**

| 周 | 里程碑 | 验证 |
|----|--------|------|
| 23 | 捕获并打印原始字节 | 能看到十六进制输出 |
| 24 | 解析 IP 头，显示源/目的地址 | 与 tcpdump 对比验证 |
| 25 | 解析 TCP/UDP，显示端口 | 能识别 HTTP/DNS 流量 |
| 26 | 过滤 + 格式化输出 + 包对比 | 按端口过滤正确；两次抓包结果可 diff |

### Fuzzing 实践（第 27 周）⭐ 安全测试技能
**做什么：** 用 AFL 对你自己写的密码工具和协议解析器做模糊测试
**学到什么：** 自动化发现 C 代码中的崩溃、越界、未定义行为
**与工作的关联：** 这个技能可以直接用于测试安全/密码设备的协议实现
```bash
sudo apt install afl++
# 用 afl-gcc 重新编译你的密码工具，然后 fuzz
afl-fuzz -i testcases/ -o findings/ ./my_crypto_tool @@
```

### Python-C 桥接（第 28 周）⭐ 连接已有技能
**做什么：** 用 `ctypes` 从 Python 调用你用 C 写的加密库
**学到什么：** 共享库（.so）的编译和加载、ABI 接口
**与工作的关联：** 你现有的 Python 测试脚本可以调用 C 实现的高性能模块
```python
import ctypes
lib = ctypes.CDLL("./libcrypto.so")
lib.xor_encrypt(data, key, length)
```

### 源码阅读（第 29-30 周）

**由浅入深，三步走：**

| 阶段 | 目标 | 行数 | 怎么读 |
|------|------|------|--------|
| 1 | Linux coreutils 的 `yes.c` | ~10 行 | 完全读懂每一行 |
| 2 | Tinyhttpd | ~500 行 | 从 `main()` 开始，用 gdb 打断点跟执行流，画调用关系图 |
| 3 | Redis 数据结构（sds.c, dict.c） | ~1000 行 | 只读数据结构部分，看工业级 C 怎么组织代码 |

**阅读方法：**
1. 先编译跑起来
2. 用 `gdb` 在 `main` 打断点，`step` 跟踪执行流
3. 画出函数调用关系图（纸上或用工具）
4. 尝试改一个小功能（如给 tinyhttpd 加一个新路由）

## 学习节奏与习惯

### 每日（1-2 小时）
```
30 分钟：看书/视频（理论）
60 分钟：写代码/做实验（实践）
10 分钟：记笔记（遇到的坑、理解的突破点）
```

### 每周检查
- [ ] 本周代码能编译通过，无警告
- [ ] 每个函数都有 assert 测试
- [ ] 用 valgrind 检查过内存
- [ ] 代码已提交 git
- [ ] 能用自己的话解释本周核心概念

### 遇到困难时
1. 自己查 `man` 手册（如 `man 3 malloc`）
2. 画内存图（纸上画格子，每个格子一个字节）
3. 用 pythontutor.com/c.html 可视化执行
4. 用 gdb 单步调试看到底发生了什么
5. 搜索或问 AI

### 卡住了怎么办
- **卡住 1 天：** 换个更简单的例子重新理解同一个概念
- **卡住 3 天：** 退回上一周的内容，确认基础是否牢固
- **卡住 1 周：** 暂停当前内容，用一个有趣的小项目恢复信心（比如用 C 写个猜数字游戏）
- **指针阶段卡住：** 这是正常的。不设时间限制，彻底搞懂再走

## 不急着学的（避免弯路）

- ❌ GoF 设计模式 —— 留给 OOP 语言
- ❌ C++ —— 先把 C 吃透
- ❌ 图形界面编程 —— 与你的目标无关
- ❌ 嵌入式/单片机 —— 除非工作需要
- ❌ 过度追求算法竞赛题 —— 重点是系统理解，不是刷题

## 里程碑总览

```
第 1-8 周   ▎基础语法 + 工具链           → 能写 200 行以内的 C 程序，无内存泄漏
第 9-16 周  ▎实战项目 + 穿插 CSAPP       → 有 3 个可运行的项目，理解字节/进程/网络
第 17-22 周 ▎CSAPP 系统章节 + 实验       → 理解汇编/链接/虚拟内存，完成 Bomb Lab
第 23-30 周 ▎综合项目 + Fuzz + 源码阅读  → 有一个与工作直接相关的工具，能读懂开源 C 代码
```

## 学完之后你会获得什么

1. **看懂底层：** 网络设备、密码设备里跑的 C 代码不再是黑箱
2. **测试更深：** 能从内存、协议、系统调用层面设计测试用例
3. **排查更准：** 用 gdb/strace/valgrind 定位问题，不再只靠日志猜
4. **安全测试：** 能用 fuzzing 发现协议实现中的漏洞
5. **Python 更强：** 能用 ctypes 让 Python 调用 C，知道性能瓶颈在哪
6. **职业天花板提高：** 安全测试、协议分析、性能测试都需要这些知识
