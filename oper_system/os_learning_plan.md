### 系统架构权衡（贯穿全课程）

Linux 选择宏内核（Monolithic Kernel）而不是微内核（Microkernel）：

- **Problem**：进程切换有成本，但需要内核帮应用程序做很多事
- **Tradeoffs**：宏内核快，但内核模块 bug 可能影响整个系统（关键路径上的 bug 会导致 kernel panic，非关键模块可能只是 oops）；微内核安全隔离好，但跨模块通信开销大
- **Limitation**：关键路径的内核 bug → kernel panic（非关键模块的 bug 可能只触发 oops，系统继续运行）

## 阶段 1：计算机启动 ✅ 已完成

### 核心理解

1. 启动是"接力"过程：BIOS → Bootloader → 内核 → init
2. 每一步都有历史包袱
3. 内核是核心，管理一切资源
4. init (PID 1) 是第一个用户进程

### 启动序列

```
电源按钮
  → BIOS/UEFI 自检
    → Bootloader（加载内核到内存）
      → 内核初始化
        → init/PID 1 (systemd)
          → 系统服务 → 登录界面
```

### 典型数值

| 操作 | 典型耗时 |
|------|---------|
| BIOS 自检 | 1-5 秒 |
| 内核加载 | 1-3 秒 |
| 内核初始化 | 1-2 秒 |
| 第一个用户进程启动 | 0.1-0.5 秒 |

### 破坏性实验

```bash
dmesg | head -50
```

### 参考来源

- 《深入理解计算机系统》— 第 8 章
- Linux 源码：`arch/x86/boot/`

## 阶段 5：中断 + 软中断 + 定时器

### 上节课回顾

上节课学了系统调用是程序主动请内核帮忙。**这节课问：有时候内核需要"插队"处理紧急事件，怎么做到的？**

### 核心问题

外设和定时器怎么通知内核？

### 将包含的内容

- 硬件中断（IRQ）机制
- **定时器中断** — 调度器时间片的物理基础
- 软中断（softirq）
- 为什么用软中断而不是硬件中断直接处理？

### 设计决策：为什么需要定时器中断？

- **Problem**：没有定时器中断，进程一旦开始运行，除非主动让出 CPU，否则会一直运行
- **Solution**：硬件定时器每隔一段时间（如 1ms）产生中断，强制把 CPU 控制权交回内核
- **这就是"时间片"的物理基础**

### 破坏性实验

```bash
# 观察硬件中断
cat /proc/interrupts

# 观察软中断
cat /proc/softirqs

# 观察定时器中断频率
getconf CLK_TCK
```

### 调试场景

"程序响应慢，怎么判断是中断处理的问题？"

## 阶段 8：同步原语

### 上节课回顾

上节课学了调度器如何决定哪个进程先跑。**这节课问：多个进程可能同时访问同一个资源，怎么办？**

### 核心问题

多进程竞争资源怎么办？

### 将包含的内容

- mutex（互斥锁）
- spinlock（自旋锁）
- 信号量
- RCU（Read-Copy-Update）
- 死锁是什么，怎么避免

### 破坏性实验

**注意：C 代码是"用 C 验证 OS 概念"的工具，不是学 C 编程**

```c
// 验证死锁现象
// 编译：gcc -o deadlock deadlock.c -lpthread
// 运行后观察：进程 hang 住，ps 显示状态是 Sl

#include <pthread.h>
#include <unistd.h>

pthread_mutex_t m1 = PTHREAD_MUTEX_INITIALIZER;
pthread_mutex_t m2 = PTHREAD_MUTEX_INITIALIZER;

void *t1(void *arg) {
    pthread_mutex_lock(&m1);
    sleep(1);  // 让另一个线程有机会抢锁
    pthread_mutex_lock(&m2);  // 卡住
}

void *t2(void *arg) {
    pthread_mutex_lock(&m2);
    sleep(1);
    pthread_mutex_lock(&m1);  // 卡住，形成死锁
}

int main() {
    pthread_t a, b;
    pthread_create(&a, NULL, t1, NULL);
    pthread_create(&b, NULL, t2, NULL);
    pthread_join(a, NULL);
    pthread_join(b, NULL);
}
```

**⚠️ 退出方法：** 运行后进程会 hang 住（这就是死锁），用 `Ctrl+C` 终止，或在另一个终端用 `kill <PID>` 杀掉。

### 调试场景

"进程 hang 住了，怎么判断是死锁？"

### 参考来源

- 《操作系统概念》— 同步章节
