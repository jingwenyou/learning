# 03 自动化安全扫描 —— 效率提升

> 手工测试深入但慢，自动化扫描快但浅。
> 最好的方式是**自动化做覆盖，手工做深入**。

---

## 一、扫描工具分类

### 1.1 DAST（动态应用安全测试）

```
原理：像攻击者一样，从外部发请求测试运行中的应用

工具：
  OWASP ZAP（免费开源，推荐新手使用）
  Burp Suite Scanner（Professional版付费功能）
  Nikto（Web服务器扫描）
  nuclei（基于模板的漏洞扫描，社区驱动）

特点：
  ✓ 不需要源码
  ✓ 测试真实运行环境
  ✗ 有误报
  ✗ 覆盖率依赖爬虫能力
  ✗ 对需要登录的功能测试困难
```

### 1.2 SAST（静态应用安全测试）

```
原理：分析源代码，找出潜在漏洞

工具：
  Semgrep（免费开源，推荐）
  SonarQube（社区版免费）
  Bandit（Python专用）
  ESLint security plugin（JavaScript）

特点：
  ✓ 覆盖率高（分析所有代码路径）
  ✓ 开发早期就能发现问题
  ✗ 误报率高
  ✗ 需要源码访问权限
  ✗ 不能发现运行时/配置问题
```

### 1.3 SCA（软件成分分析）

```
原理：检查项目依赖库是否有已知漏洞

工具：
  npm audit（Node.js自带）
  pip-audit（Python）
  OWASP Dependency-Check（通用）
  Snyk（免费版可用）
  Trivy（容器镜像+依赖扫描）

特点：
  ✓ 自动化程度高
  ✓ 直接关联CVE数据库
  ✗ 有已知漏洞不代表可利用
```

## 二、OWASP ZAP快速上手

```
安装：
  https://www.zaproxy.org/download/
  或 Docker: docker run -u zap -p 8080:8080 ghcr.io/zaproxy/zaproxy

基本使用：
  1. 自动扫描：输入URL → 点Attack → 等待结果
  2. 手动探索：配置代理（类似Burp）→ 手动浏览 → 再扫描
  3. API扫描：导入OpenAPI/Swagger文档 → 扫描所有接口

结果分析：
  - High：高危，优先处理（SQL注入、XSS等）
  - Medium：中危，需要评估
  - Low：低危，建议修复
  - Info：信息性发现

每个发现都要人工验证——不要直接把扫描报告当成漏洞报告。
```

## 三、将扫描融入测试流程

```
推荐流程：

[1] 先手工 → 理解应用，建立攻击面地图
[2] 再自动化 → 用扫描器做广度覆盖
[3] 回手工 → 对扫描器报告的可疑项做人工验证
[4] 写报告 → 只报经过验证的漏洞

扫描器的角色是"助手"，不是"替代"。
它帮你发现你可能遗漏的点，但最终判断靠你。
```
