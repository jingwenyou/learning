# IPsec 多隧道并发压测工具

## 概述

本工具用于测试 IPsec 多隧道场景下的并发收发包能力，支持从多个源IP同时向目标IP发送数据包，并统计延迟、丢包率和吞吐量。

## 组件

| 组件 | 位置 | 用途 |
|------|------|------|
| `ipsec_multi_tunnel_test.py` | 192.168.110.240 | 发包测试工具（客户端） |
| `ipsec_echo_server.py` | 192.168.110.199 | 回包服务（服务端） |

## 快速开始

### 1. 启动回包服务（192.168.110.199）

```bash
ssh haitiai@192.168.110.199
python3 /tmp/ipsec_echo_server.py
```

或后台运行：

```bash
ssh haitiai@192.168.110.199 "nohup python3 /tmp/ipsec_echo_server.py > /tmp/echo_server.log 2>&1 &"
```

### 2. 运行压测（192.168.110.240）

```bash
python3 /tmp/ipsec_multi_tunnel_test.py -d 10.3.3.199 -i enp1s0f0 -r 100
```

按 `Ctrl+C` 停止测试。

## 发包工具参数说明

```bash
python3 ipsec_multi_tunnel_test.py [参数]
```

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `-d DST` | 目标IP | 10.3.3.199 |
| `-i IFACE` | 源网卡，从该网卡获取所有IP | enp1s0f0 |
| `-t TIME` | 持续时间（秒），0=无限 | 0 |
| `-r RATE` | 每IP每秒发包数 | 100 |
| `-s SIZE` | 数据包大小（字节） | 1024 |

### 使用示例

```bash
# 基础测试：每IP 100pps，持续运行
python3 ipsec_multi_tunnel_test.py -d 10.3.3.199 -i enp1s0f0 -r 100

# 限时测试：运行1小时，每IP 200pps
python3 ipsec_multi_tunnel_test.py -d 10.3.3.199 -i enp1s0f0 -t 3600 -r 200

# 大包测试：512字节
python3 ipsec_multi_tunnel_test.py -d 10.3.3.199 -i enp1s0f0 -r 100 -s 512
```

## 原理说明

### 架构

```
┌─────────────────────────┐                    ┌─────────────────────────┐
│   192.168.110.240       │                    │   192.168.110.199       │
│  ┌───────────────────┐  │                    │  ┌───────────────────┐  │
│  │ ipsec_multi_      │  │                    │  │ ipsec_echo_       │  │
│  │ tunnel_test.py    │  │ ════════════════>  │  │ server.py         │  │
│  │                   │  │   IPsec 隧道       │  │                   │  │
│  │ [Worker x 64]    │  │ <════════════════  │  │ [回包服务]        │  │
│  │ 每Worker:        │  │   UDP Echo         │  │ 监听 7000-7063   │  │
│  │  send + recv    │  │                    │  │                   │  │
│  └───────────────────┘  │                    │  └───────────────────┘  │
└─────────────────────────┘                    └─────────────────────────┘
```

### 测试流程

1. **发包**: 每个源IP启动一个 Worker（send线程 + recv线程共用一个socket）
2. **回包**: 对端 echo_server 收到包后原路返回
3. **确认应答**: 发送后超过5秒未收到回包才计为丢失
4. **延迟计算**: 仅统计5秒内返回的包，避免严重延迟/乱序包拉高平均值

### 丢包判断逻辑

- **待返(途中)**: 包已发出但尚未返回，属于正常在途
- **丢失**: 超过5秒未收到应答，标记为真正丢失
- **丢包率**: `lost / (received + lost)`，仅基于已确认的包计算

### 字段说明

| 字段 | 说明 |
|------|------|
| 发送 | 已发送的数据包总数 |
| 接收 | 已收到回包的数量 |
| 丢失 | 超过5秒超时确认丢失的包数 |
| 丢包率 | lost / (received + lost) × 100% |
| 延迟AVG | 正常回包（<5s）的平均往返延迟 |
| 延迟P99 | 正常回包的P99延迟 |
| 待返(途中) | 已发出但尚未返回的包（正常现象） |

---

## 判断标准

| 丢包率 | 网络状态 |
|--------|---------|
| ≈ 0% | 网络良好 |
| 0.1% - 1% | 基本正常，可能有轻微问题 |
| > 1% | 可能存在丢包或延迟过高 |

| 延迟 | 状态 |
|------|------|
| < 10ms | 极好，同城IDC |
| 10-50ms | 正常，可接受 |
| 50-200ms | 偏高，可能经过较多跳转 |
| > 200ms | 可能存在隧道拥塞或设备瓶颈 |
